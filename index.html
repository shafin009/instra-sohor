Javascript একটা Interpreted Programming Language.এটি প্রাথমিকভাবে দুটি ভিন্ন উদ্দেশ্যে ব্যবহৃত হয়। প্রথমত Client Side প্রোগ্রামিং Dynamic Web Page তৈরি করতে আর দ্বিতীয়ত Server Side প্রোগ্রামিং API তৈরি করতে । Javascript ইঞ্জিনগুলোর সাহায্যে Client Side Javascript কাজ করে যা ব্রাউজারগুলির পিছনে চলে। বিভিন্ন ব্রাউজার কোম্পানির লেখা বিভিন্ন ইঞ্জিন আছে।যেমনঃ-Internet Explorer এর Chakra, Firefox এর Spidermonkey, Chrome এর V8 ইত্যাদি। বর্তমানে Server Side এ আমরা node.js ব্যবহার করি যা V8 ইঞ্জিনের উপর একটি বর্ধিতকরণ এবং API তৈরি এবং Database এর সাথে কথা বলার মতো Server Side প্রোগ্রামিংয়ের জন্য Javascript চালাতে ব্যবহৃত হয়।

Event Loop হল JavaScript এর asynchronous programming এর ভেতরকার রহস্য। Event Loop হল একটি প্রক্রিয়া যা task queue থেকে call stack এ callback কে ঠেলে দেওয়ার আগে call stack পরিষ্কার হওয়ার জন্য অপেক্ষা করে। Stack পরিষ্কার হয়ে গেলে, event loop trigger করে । উপলব্ধ callback এর জন্য task queue তা চেক করে। যদি কোনোটি থাকে তবে এটি call stack এর দিকে ঠেলে দেয় । আবার পরিষ্কার হওয়ার জন্য call stack অপেক্ষা করে এবং একই প্রক্রিয়া চলমান থাকে।

Local storage এ data স্থায়ীভাবে থাকে। Session storage এ data সাময়িকভাবে থাকে। Local storage ও Session storage থেকে কোনো নির্দিষ্ট মান return করা হয় getItem() ব্যবহার করে। storage.setItem(KeyName,KeyValue) ব্যবহার করে Local storage এবং Session storage এ মান সেট করা হয়। Local storage ও Session storage এর object নিয়ে কাজ করতে হলে সেটাকে প্রথমে JSON.parse() করতে হবে। Local storage ও Session storage এ কোনো object সেট করার আগে JSON.stringify() করতে হবে। Browser কে বন্ধ করে দিলে Session storage এর সব data মুছে যাবে কিন্তু Local storage এর data মুছে যাবে না।
